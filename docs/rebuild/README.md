# Rebuild Workspace

This directory is the working documentation space for the Modal -> Vercel Sandbox rebuild.

## Purpose

- Keep a durable record of what we decided, what changed, and why.
- Track execution progress against the approved phase plan.
- Capture architecture choices that are safe to revisit later.

## Working Rules

- Update `progress-log.md` whenever a to-do moves to complete, blocked, or changed.
- Record all material architecture decisions as ADRs in `adrs/`.
- Snapshot significant plan revisions into `archive/` with date/version naming.
- Keep checklists aligned with shipped code, not intent.
